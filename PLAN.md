# 像素风 RPG 战棋网页游戏 — 规划文档

> 最后更新: 2026-06-27

---

## 一、游戏定位

| 项目 | 内容 |
|------|------|
| 游戏类型 | 回合制战棋 + RPG 剧情 |
| 核心玩法 | PVE 剧情关卡，局外养成角色，局内回合制战斗 |
| 单局时长 | 无限制（剧情驱动） |
| 目标平台 | Web 浏览器（PC） |
| 部署方式 | 本地运行（localhost） |

---

## 二、战斗系统

### 2.1 回合机制
- 双方交替行动：玩家全队行动 → 敌方全队行动
- 每方回合内，每个单位可执行一次 **移动 + 一次行动**（攻击/使用技能/待命）

### 2.2 队伍规模
- 每方 3~5 个单位

### 2.3 概率机制
- **命中率**：攻击未必命中，受双方属性影响
- **暴击率**：命中后概率触发，伤害 x1.5（暂定）
- **闪避率**：概率完全规避伤害

### 2.4 伤害公式（简单加减）
```
物理伤害 = 攻击力 - 防御力 + 随机偏移(0~3)
魔法伤害 = 魔法攻击 - 魔法防御 + 随机偏移(0~3)
暴击时 × 1.5
未命中 / 闪避时伤害为 0
最低伤害保底 = 1
```

### 2.5 地形效果

| 地形 | 效果 |
|------|------|
| **平地** | 无加成，移动消耗 1 |
| **高地** | 攻击力 +20%，远程射程 +1；移动消耗 1 |
| **水域** | 不可通行（飞行单位除外） |
| **障碍物** | 不可通行，阻挡远程攻击弹道 |

## 三、局外养成系统

| 系统 | 说明 |
|------|------|
| **等级** | 角色通过战斗获得经验升级，提升基础属性 |
| **装备** | 武器、防具、饰品三个槽位，提供属性加成 |
| **技能** | 角色可学习多个技能，战斗中消耗资源使用 |
| **升级** | 可消耗材料提升技能等级/装备强化 |

## 四、美术风格

| 项目 | 规格 |
|------|------|
| 像素粒度 | 32px（tile 尺寸 32×32） |
| UI风格 | 像素 + 现代 UI 混搭 |
| 动画帧率 | 每秒 24 帧 |
| 动画动作 | 待机 / 移动 / 攻击 / 施法 / 受击 / 死亡 |
| 地图渲染 | Tilemap + WebGL |
| 素材来源 | AI 生成 + 手工调整 |
| 帧动画方向 | 暂定仅左右方向（或四方向待定） |

## 五、技术架构

### 5.1 客户端

| 项目 | 技术选型 |
|------|----------|
| 渲染引擎 | Phaser 3（WebGL） |
| 构建工具 | Vite + TypeScript |
| 状态管理 | Phaser Scene + 自定义状态机 |
| 通信 | WebSocket 连接后端 |

### 5.2 服务端

| 项目 | 技术选型 |
|------|----------|
| HTTP 框架 | Express |
| 实时通信 | ws（WebSocket） |
| 数据库 | SQLite（better-sqlite3） |
| 运行时 | Node.js |

### 5.3 WebSocket 用途
- 实时自动存档（战斗进度即时写入服务端）
- 服务端权威校验（防止客户端篡改数据）
- 未来扩展：异步观战 / 回放

## 六、MVP （Minimum Viable Product）

### 6.1 MVP 目标
> **在一张固定地图上，完成玩家 vs AI 的回合制战棋战斗**

### 6.2 MVP 功能清单

- [x] 启动项目（前端 + 后端）
- [ ] 加载固定 Tilemap 地图（包含平地/高地/水域/障碍物）
- [ ] 玩家控制队伍 3 个单位，敌方队伍 3 个单位
- [ ] 回合交替系统（玩家全队 → AI全队）
- [ ] 单位移动（点击目标格子 → 角色走到目标位置）
- [ ] 攻击逻辑（选择目标 → 命中/暴击/闪避计算 → 伤害）
- [ ] 手动交替攻击（每个单位独立选择攻击目标）
- [ ] 简单 AI（敌方自动寻敌攻击）
- [ ] 战斗结算（胜负判定）
- [ ] 自动存档（WebSocket 实时存 SQLite）
- [ ] 像素动画（待机、移动、攻击各 24fps）

### 6.3 MVP 不包括（后续迭代）
- 养成系统（等级/装备/技能界面）
- 剧情对话系统
- 多关卡/地图选择
- 角色招募/编队界面
- 音效与音乐

## 九、技术决策记录

| 日期 | 决策 | 理由 |
|------|------|------|
| 2026-06-27 | 选中 Phaser 3 作为渲染引擎 | 成熟 2D 游戏框架，WebGL 支持好，社区活跃 |
| 2026-06-27 | 选中 SQLite + better-sqlite3 | 单机级持久化，无需额外服务部署 |
| 2026-06-27 | 即使单机 PVE 也用 WebSocket | 服务端权威校验 + 自动存档，防篡改 |
| 2026-06-27 | 伤害公式采用简单加减 | MVP 阶段快速实现验证，后期可调整 |
| 2026-06-27 | 角色动画 24fps | 平衡像素美术精细度与性能 |

## 十、风险与待定项

### 风险
1. **AI 生成像素图一致性**：不同动作/方向的角色图需要风格统一 → 需要后期手工调整
2. **Phaser 3 + TypeScript 类型定义**：社区类型定义不够完善 → 需要自定义类型
3. **服务端校验 + 客户端表现同步**：简单加减计算量小，风险可控

### 待定项
- [ ] 角色动画的方向：仅左右 vs 四方向（上下左右）
- [ ] 技能系统具体设计（消耗资源/冷却/范围）
- [ ] 装备属性数值体系
- [ ] 经验/升级曲线
- [ ] 剧情脚本格式
